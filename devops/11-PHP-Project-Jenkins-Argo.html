TASK 11: PHP PROJECT, PIPELINE FOR JENKINS CI AND ARGO CD

  1. Prerequisite Installation

    The necessary tools were installed on the virtual machine, including:

    • Git for version control
    • Docker for containerization
    • PHP for local testing of the application
    • Jenkins for implementing continuous integration
    • Kubernetes (e.g., via Minikube or Kind) for container orchestration
    • Argo CD for GitOps-based continuous deployment

  2. Project Initialization

    A new project directory was created (php-hello-world) containing:

    • index.php: A basic PHP script displaying "Hello, World!"
    • Dockerfile: Configuration for building a PHP-Apache container image
    • README.md: A project description file
    • Directory structure for CI/CD (jenkins/) and deployment manifests (k8s/)

  3. GitHub Integration

    The local project was initialized as a Git repository. A new GitHub repository was created, and the local code was pushed to the remote repository.

  4. Jenkins Pipeline Configuration

    A Jenkinsfile was created to define the CI/CD pipeline. The stages included:

    • Cloning the repository
    • Building the Docker image
    • Pushing the image to Docker Hub
    • Placeholder for triggering an Argo CD deployment update

  5. Kubernetes Deployment Manifests

    Kubernetes manifests were created under the k8s/ directory:

    • A Deployment to run the PHP container
    • A Service to expose the application on port 80

    These manifests specify how the application should be run within the Kubernetes cluster.

  6. Argo CD Setup

    Argo CD was installed in the Kubernetes cluster. An Application resource was defined to:

    • Automatically synchronize the Kubernetes manifests from the GitHub repository
    • Continuously monitor and apply changes using GitOps principles

  7. Jenkins Configuration

    Jenkins was configured to:

    • Use the GitHub repository as the source
    • Authenticate with Docker Hub using credentials
    • Run the pipeline defined in jenkins/Jenkinsfile

  8. Continuous Integration and Deployment

    A Jenkins build was triggered, which:

    • Built the Docker image
    • Pushed it to Docker Hub
    • Enabled Argo CD to detect the image change and deploy it to the cluster

Summary

This project establishes a full DevOps pipeline:

  • The source code is maintained in GitHub.
  • Jenkins handles continuous integration by building and pushing Docker images.
  • Kubernetes runs the containerized application.
  • Argo CD performs automated, Git-driven continuous deployment.

This architecture follows modern DevOps and GitOps best practices for scalable, automated application delivery.
