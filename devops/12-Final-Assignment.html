TASK 12: FINAL ASSIGNMENT

The DevOps Internship Final Assignment is a comprehensive, hands-on project designed to assess practical knowledge of modern DevOps methodologies. The objective is to design and implement a complete CI/CD pipeline for a simple web application using industry-standard tools. The assignment is divided into five key stages:

  • Web Application Development – A basic web application is developed using Flask that returns a "Hello from DevOps" message, structured with proper files and hosted on GitHub.
  • Docker Containerization – The application is containerized using Docker by writing a Dockerfile and exposing the appropriate port, showcasing understanding of image creation and container management.
  • CI Pipeline with GitHub Actions – A continuous integration (CI) pipeline is implemented using GitHub Actions, which triggers on code pushes, installs dependencies, runs basic tests, and builds the Docker image.
  • Kubernetes Deployment – The containerized application is deployed to a local Kubernetes cluster (Minikube), with appropriate manifests for deployment and service, ensuring accessibility of the application via the cluster.
  • Documentation – A detailed README file is created, explaining the tools used, setup steps, and the CI/CD workflow, supported with relevant screenshots.

This assignment provides practical exposure to essential DevOps tools such as Git, Docker, GitHub Actions, and Kubernetes, emphasizing automation, reproducibility, and scalable deployment in modern software development workflows.

Step-by-Step Implementation

Step 1: Web Application Development

  • A simple web application was developed using the Flask framework in Python.
  • The application was designed to respond with a basic message, “Hello from DevOps”, when accessed via a web browser.
  • The project was organized following standard best practices, with separate files for application logic (app.py) and dependencies (requirements.txt).
  • The complete source code was version-controlled using Git and hosted on GitHub for collaborative access and integration with CI/CD tools.

Step 2: Docker Containerization

  • A Dockerfile was created to containerize the Flask application, ensuring it could be executed consistently across different environments.
  • The Docker image was configured to expose port 5000, which is the default for Flask applications.
  • The container was tested locally using Docker CLI commands to verify successful build and execution.
  • Optionally, the image was tagged and pushed to a Docker Hub repository to demonstrate proficiency in container registry usage.

Step 3: Continuous Integration with GitHub Actions

  • A CI pipeline was configured using GitHub Actions.
  • A YAML workflow file (ci.yml) was created under .github/workflows/ to automate tasks such as:
    • Triggering on push events to the main branch
    • Installing Python dependencies
    • Running basic tests (e.g., syntax or dummy checks)
    • Building the Docker image
  • This step ensured that every code update was automatically validated, promoting development consistency and early error detection.

Step 4: Kubernetes-Based Deployment

  • A local Kubernetes cluster was set up using Minikube to simulate real-world deployment scenarios.
  • Kubernetes manifest files were written:
    • deployment.yaml defined the application’s deployment strategy and container configuration.
    • service.yaml exposed the application using a NodePort service to allow external access.
  • The deployment was validated using kubectl commands, confirming that the pod was running and the application was accessible via browser or port-forwarding.

Step 5: Documentation and Reporting

  • A comprehensive README.md file was prepared, documenting the project’s purpose, structure, tools used, setup instructions, and the CI/CD workflow.
  • Screenshots were included to visually demonstrate:
    • The running application
    • Successful CI pipeline execution
    • Kubernetes pod status
    • Application accessibility from the cluster
  • This step emphasized clarity, reproducibility, and alignment with DevOps communication practices.
